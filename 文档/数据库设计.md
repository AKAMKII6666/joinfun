# 组局小程序 - 数据库设计文档

## 一、设计概述

**数据库类型**：MySQL / PostgreSQL（推荐 MySQL 8.0+）  
**字符集**：utf8mb4  
**设计原则**：简洁、高效、易扩展

---

## 二、数据表结构

### 2.1 用户表 `users`

存储小程序用户信息。

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | ✅ | AUTO_INCREMENT | 主键 |
| openid | VARCHAR | 64 | ✅ | - | 微信 openid（唯一索引） |
| unionid | VARCHAR | 64 | - | NULL | 微信 unionid（可选） |
| nickname | VARCHAR | 50 | ✅ | - | 昵称 |
| avatar_url | VARCHAR | 255 | - | NULL | 头像 URL |
| modify_count | INT | - | ✅ | 0 | 年内修改次数 |
| last_modify_time | DATETIME | - | - | NULL | 最后修改时间 |
| created_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_openid (openid)

**说明**：
- `modify_count` 每年 1 月 1 日清零（可通过定时任务或逻辑判断实现）
- 昵称和头像年内修改限制 3 次

---

### 2.2 活动表 `activities`

存储活动信息。

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | ✅ | AUTO_INCREMENT | 主键 |
| user_id | BIGINT | - | ✅ | - | 发起人 ID（外键→users.id） |
| title | VARCHAR | 40 | ✅ | - | 活动标题 |
| cover_image | VARCHAR | 255 | - | NULL | 封面图 URL |
| max_participants | INT | - | ✅ | 999 | 限制人数 |
| location_name | VARCHAR | 100 | - | NULL | 地点名称 |
| location_address | VARCHAR | 255 | - | NULL | 详细地址 |
| latitude | DECIMAL | 10,8 | - | NULL | 纬度 |
| longitude | DECIMAL | 11,8 | - | NULL | 经度 |
| contact_phone | VARCHAR | 11 | - | NULL | 联系电话 |
| description | TEXT | - | - | NULL | 活动说明（1000 字内） |
| detail_images | JSON | - | - | NULL | 详情图片数组（上限 12 张） |
| guide_images | JSON | - | - | NULL | 引路图片数组（上限 12 张） |
| status | TINYINT | - | ✅ | 1 | 状态：1-进行中，2-已取消，3-已结束 |
| view_count | INT | - | ✅ | 0 | 浏览人数 |
| registration_count | INT | - | ✅ | 0 | 报名人数 |
| created_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- KEY idx_user_id (user_id)
- KEY idx_status (status)
- KEY idx_created_at (created_at)

---

### 2.3 活动日期表 `activity_dates`

存储活动的多个日期时间（一个活动可有多个日期）。

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | ✅ | AUTO_INCREMENT | 主键 |
| activity_id | BIGINT | - | ✅ | - | 活动 ID（外键→activities.id） |
| date | DATE | - | ✅ | - | 活动日期 |
| start_time | TIME | - | ✅ | - | 开始时间 |
| end_time | TIME | - | ✅ | - | 结束时间 |
| created_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- PRIMARY KEY (id)
- KEY idx_activity_id (activity_id)

---

### 2.4 报名表 `registrations`

存储用户报名关系。

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | ✅ | AUTO_INCREMENT | 主键 |
| activity_id | BIGINT | - | ✅ | - | 活动 ID（外键→activities.id） |
| user_id | BIGINT | - | ✅ | - | 用户 ID（外键→users.id） |
| status | TINYINT | - | ✅ | 1 | 状态：1-已报名，2-已请假 |
| created_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP | 报名时间 |
| updated_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_activity_user (activity_id, user_id) - 防止重复报名
- KEY idx_user_id (user_id)
- KEY idx_activity_id (activity_id)

---

### 2.5 浏览记录表 `view_logs`

存储用户浏览记录（用于"最近浏览"功能）。

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | ✅ | AUTO_INCREMENT | 主键 |
| user_id | BIGINT | - | ✅ | - | 用户 ID（外键→users.id） |
| activity_id | BIGINT | - | ✅ | - | 活动 ID（外键→activities.id） |
| viewed_at | DATETIME | - | ✅ | CURRENT_TIMESTAMP | 浏览时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_user_activity (user_id, activity_id) - 同一用户同一活动只保留一条
- KEY idx_user_id (user_id)
- KEY idx_viewed_at (viewed_at)

## 四、SQL 初始化脚本

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS joinfun DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE joinfun;

-- 用户表
CREATE TABLE `users` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `openid` VARCHAR(64) NOT NULL,
  `unionid` VARCHAR(64) DEFAULT NULL,
  `nickname` VARCHAR(50) NOT NULL,
  `avatar_url` VARCHAR(255) DEFAULT NULL,
  `modify_count` INT NOT NULL DEFAULT 0,
  `last_modify_time` DATETIME DEFAULT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_openid` (`openid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';

-- 活动表
CREATE TABLE `activities` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `title` VARCHAR(40) NOT NULL,
  `cover_image` VARCHAR(255) DEFAULT NULL,
  `max_participants` INT NOT NULL DEFAULT 999,
  `location_name` VARCHAR(100) DEFAULT NULL,
  `location_address` VARCHAR(255) DEFAULT NULL,
  `latitude` DECIMAL(10,8) DEFAULT NULL,
  `longitude` DECIMAL(11,8) DEFAULT NULL,
  `contact_phone` VARCHAR(11) DEFAULT NULL,
  `description` TEXT DEFAULT NULL,
  `detail_images` JSON DEFAULT NULL,
  `guide_images` JSON DEFAULT NULL,
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '1-进行中，2-已取消，3-已结束',
  `view_count` INT NOT NULL DEFAULT 0,
  `registration_count` INT NOT NULL DEFAULT 0,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_status` (`status`),
  KEY `idx_created_at` (`created_at`),
  CONSTRAINT `fk_activities_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='活动表';

-- 活动日期表
CREATE TABLE `activity_dates` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `activity_id` BIGINT NOT NULL,
  `date` DATE NOT NULL,
  `start_time` TIME NOT NULL,
  `end_time` TIME NOT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_activity_id` (`activity_id`),
  CONSTRAINT `fk_activity_dates_activity` FOREIGN KEY (`activity_id`) REFERENCES `activities` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='活动日期表';

-- 报名表
CREATE TABLE `registrations` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `activity_id` BIGINT NOT NULL,
  `user_id` BIGINT NOT NULL,
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '1-已报名，2-已请假',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_activity_user` (`activity_id`, `user_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_activity_id` (`activity_id`),
  CONSTRAINT `fk_registrations_activity` FOREIGN KEY (`activity_id`) REFERENCES `activities` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_registrations_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='报名表';

-- 浏览记录表
CREATE TABLE `view_logs` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `activity_id` BIGINT NOT NULL,
  `viewed_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_user_activity` (`user_id`, `activity_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_viewed_at` (`viewed_at`),
  CONSTRAINT `fk_view_logs_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_view_logs_activity` FOREIGN KEY (`activity_id`) REFERENCES `activities` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='浏览记录表';
```

## 五、核心查询示例

### 5.1 获取用户已报名活动列表

```sql
SELECT 
  a.id, a.title, a.cover_image, a.location_name,
  ad.date, ad.start_time, ad.end_time,
  a.registration_count
FROM activities a
JOIN registrations r ON a.id = r.activity_id
JOIN activity_dates ad ON a.id = ad.activity_id
WHERE r.user_id = ? AND r.status = 1
ORDER BY 
  CASE WHEN ad.date < CURDATE() THEN 1 ELSE 0 END,
  ad.date ASC, ad.start_time ASC;
```

### 5.2 获取用户最近浏览活动

```sql
SELECT 
  a.id, a.title, a.cover_image, a.location_name,
  ad.date, ad.start_time, ad.end_time,
  a.registration_count, vl.viewed_at
FROM view_logs vl
JOIN activities a ON vl.activity_id = a.id
JOIN activity_dates ad ON a.id = ad.activity_id
WHERE vl.user_id = ?
ORDER BY vl.viewed_at DESC
LIMIT 10;
```

### 5.3 获取用户发起的活动列表

```sql
SELECT 
  a.id, a.title, a.cover_image, a.location_name,
  ad.date, ad.start_time, ad.end_time,
  a.registration_count, a.status
FROM activities a
JOIN activity_dates ad ON a.id = ad.activity_id
WHERE a.user_id = ?
ORDER BY a.created_at DESC;
```

### 5.4 获取活动详情及报名状态

```sql
SELECT 
  a.*, u.nickname AS organizer_nickname, u.avatar_url AS organizer_avatar,
  CASE WHEN r.id IS NOT NULL THEN r.status ELSE 0 END AS my_registration_status
FROM activities a
JOIN users u ON a.user_id = u.id
LEFT JOIN registrations r ON a.id = r.activity_id AND r.user_id = ?
WHERE a.id = ?;
```

## 六、注意事项

1. **图片存储**：所有图片 URL 建议存储到对象存储（如阿里云 OSS、腾讯云 COS），数据库只存 URL
2. **微信登录**：需要后端实现微信登录接口，获取 openid 后自动注册/登录
3. **修改次数限制**：年内修改 3 次的逻辑需要在后端实现（判断 `last_modify_time` 是否跨年）
4. **活动状态**：`status` 字段可通过定时任务更新（过期活动自动标记为已结束）
5. **并发控制**：报名时的名额限制需要加锁或使用乐观锁防止超卖
