# 组局小程序 - 设计导向稿

> 用于对接第三方设计资源，详细说明每个页面的所有元素和交互细节
> 版本：1.0 | 基于需求说明书整理

---

## 一、整体结构

### 1.1 页面架构

| 页面 | 路径 | 说明 |
|------|------|------|
| 首页（我参与的） | / | 默认首页，展示已报名活动 + 最近浏览 |
| 活动详情 | /activity/:id | 活动详细信息 + 报名 |
| 我发起的 | /my-activities | 我发布的活动列表 |
| 发布新活动 | /publish | 创建新活动 |
| 编辑活动 | /edit/:id | 编辑已有活动 |
| 我的 | /profile | 个人信息 + 设置 |

### 1.2 底部导航栏

```
┌─────────────────────────────────────┐
│                                     │
│           页面内容区域               │
│                                     │
├─────────────────────────────────────┤
│  我参与的    │   我发起的    │  我的  │
│    (首页)    │              │        │
└─────────────────────────────────────┘
```

- 共 3 个 tab：首页（我参与的）、我发起的、我的
- 当前 tab 高亮显示
- 点击切换页面

## 二、详细页面设计

### 2.1 首页（我参与的 / 已报名）

#### 页面结构

```
┌─────────────────────────────────────┐
│  组局小助手                        │ ← 顶部标题栏
├─────────────────────────────────────┤
│  [已报名]  [最近浏览]             │ ← Tab 切换
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 🖼️ 封面图                  │   │
│  │ 活动名称                   │   │
│  │ 12-09 周二 14:00~16:00   │   │ ← 活动卡片
│  │ 📍 保利国际影城...          │   │
│  │ 👥 1/999 人               │   │
│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 🖼️ 封面图                  │   │
│  │ 活动名称                   │   │
│  │ ...                        │   │
│  └─────────────────────────────┘   │
│                                     │
├─────────────────────────────────────┤
│  🆓 发布                           │ ← 悬浮发布按钮
├─────────────────────────────────────┤
│  我参与的  │  我发起的  │  我的   │
└─────────────────────────────────────┘
```

#### 元素详情

| 元素 | 规格 | 交互 |
|------|------|------|
| 顶部标题 | "组局小助手"，居中 | - |
| Tab 栏 | 两个 tab：已报名、最近浏览 | 点击切换，滑动切换 |
| 活动卡片 | 封面图(16:9) + 标题 + 时间 + 地点 + 人数 | 点击进入详情 |
| 活动名称 | 最多显示2行，超出省略 | - |
| 时间 | 格式：12-09 周二 14:00~16:00 | - |
| 地点 | 商家名称，溢出省略 | - |
| 人数 | 格式：1/999 人 | - |
| 封面图 | 默认图（无图时显示占位） | - |
| 发布悬浮窗 | 右下角固定，圆形按钮 | 点击进入发布页 |

#### 空状态

- 未报名任何活动时：显示缺省页
- 提示文案："未报名活动"
- 引导按钮："免费发布"（点击进入发布页）

#### 排序规则

- **已报名 Tab**：当前日期之前的活动置底，其余正序排列
- **最近浏览 Tab**：按浏览时间倒序（最新的在最上面）

#### 元素详情

| 元素 | 规格 | 交互 |
|------|------|------|
| 返回按钮 | 左上角 | 点击返回上一页 |
| 分享按钮 | 右上角 | 点击唤起微信分享 |
| 顶部封面图 | 宽度100%，高度自适应 | - |
| 活动标题 | 最多40字 | - |
| 发起人信息 | 头像(圆形, 32x32) + 昵称 | - |
| 活动时间 | 格式：12-09 周二 14:00~16:00 | - |
| 活动地点 | 商家名称 + 详细地址 | 点击唤起地图导航 |
| 报名人数 | 格式：1/999 人 | - |
| 报名状态 | 未报名/已报名/请假中 | 根据用户状态显示不同 |
| 报名按钮 | 底部固定按钮 | 点击报名/请假 |
| 已报名人列表 | 头像+昵称，可折叠 | 点击展开/收起 |
| 活动说明 | 富文本，最长1000字 | - |
| 详情图片 | 网格排列，最多12张 | 点击查看大图 |
| 引路图片 | 报名后显示，最多12张 | 点击查看大图 |

#### 报名按钮状态

| 状态 | 按钮文字 | 颜色 |
|------|----------|------|
| 未报名 | 立即报名 | 主色调 |
| 已报名 | 请假 | 次色调 |
| 请假中 | 已请假 | 灰色 |

#### 报名流程

1. 点击"立即报名"
2. 弹窗确认（展示引路图片）
3. 确认后更新状态为"已报名"
4. 按钮变为"请假"

- 若活动时间变更，弹窗提示："活动时间变更，请通知报名人"

## 三、公共组件

### 3.1 授权弹窗

- 触发条件：未授权用户进行需授权操作
- 内容：获取昵称、头像、Openid
- 按钮：微信授权

### 3.2 分享组件

- 触发：点击分享按钮
- 方式：微信原生分享
- 分享内容：活动标题 + 封面图 + 链接

### 3.3 地图导航

- 触发：点击活动地点
- 方式：唤起微信地图/系统地图
- 功能：查看位置、导航

### 3.4 图片预览

- 触发：点击图片
- 方式：全屏预览，左右滑动
- 功能：放大、缩小、保存

### 3.5 空状态

- 样式：居中图标 + 提示文字 + 引导按钮
- 场景：未报名、未浏览、未发布
