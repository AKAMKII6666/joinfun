# 组局小程序 - 编码规范

## 一、CSS 规范

### 1.1 全局颜色变量
所有颜色必须定义在 `app.wxss` 中，使用 `var()` 引用：

```css
/* app.wxss */
page {
  /* 主色调 */
  --primary-color: #07C160;
  --primary-color-light: #09CE68;
  --primary-color-dark: #059B48;

  /* 次色调 */
  --secondary-color: #999999;
  --secondary-color-light: #B3B3B3;
  --secondary-color-dark: #808080;

  /* 背景色 */
  --bg-color: #F5F5F5;
  --bg-color-white: #FFFFFF;

  /* 文字颜色 */
  --text-main: #333333;
  --text-secondary: #666666;
  --text-placeholder: #999999;
  --text-disabled: #CCCCCC;

  /* 分割线 */
  --border-color: #E5E5E5;
  --border-color-light: #F0F0F0;

  /* 状态色 */
  --success-color: #07C160;
  --warning-color: #FF9900;
  --error-color: #FA5151;
  --info-color: #10AEFF;
}
```

### 1.2 使用方式
```css
/* ✅ 正确 */
.activity-item {
  background-color: var(--bg-color-white);
  color: var(--text-main);
  border-color: var(--border-color);
}

/* ❌ 错误 */
.activity-item {
  background-color: #FFFFFF;
  color: #333333;
}
```

## 二、JavaScript 规范

### 2.1 禁止三元表达式
使用 `if...else` 或 `switch`：

```javascript
/* ✅ 正确 */
if (status === 'loading') {
  wx.showLoading({ title: '加载中' });
} else {
  wx.hideLoading();
}

/* ❌ 错误 */
status === 'loading' ? wx.showLoading() : wx.hideLoading();
```

```javascript
/* ✅ 正确 - switch */
switch (type) {
  case 'add':
    this.handleAdd();
    break;
  case 'edit':
    this.handleEdit();
    break;
  case 'delete':
    this.handleDelete();
    break;
  default:
    break;
}
```

### 2.2 箭头函数慎用
- 只有在需要绑定 `this` 时使用箭头函数
- 其他情况使用普通函数

```javascript
/* ✅ 正确 - 需要绑定 this */
const that = this;
wx.request({
  success: (res) => {
    that.setData({ list: res.data });
  }
});

/* ✅ 正确 - 普通函数 */
Page({
  onLoad: function(options) {
    // ...
  }
});

/* ❌ 错误 - 没必要用箭头函数 */
Page({
  onLoad: (options) => {
    // ...
  }
});
```

## 三、WXML 规范

### 3.1 每个布局节点都要有注释
```html
<!-- 页面容器 -->
<view class="container">

  <!-- 顶部标题 -->
  <view class="header">
    <text class="title">活动列表</text>
  </view>

  <!-- 活动列表 -->
  <view class="activity-list">
    <block wx:for="{{activities}}">
      <!-- 活动项 -->
      <view class="activity-item">
        <image class="cover" src="{{item.cover}}" />
        <text class="name">{{item.name}}</text>
      </view>
    </block>
  </view>

  <!-- 空状态 -->
  <view wx:if="{{activities.length === 0}}" class="empty">
    <text>暂无活动</text>
  </view>

</view>
```

## 四、组件化规范

### 4.1 组件目录结构
```
components/
├── activity-card/        # 活动卡片组件
│   ├── activity-card.js
│   ├── activity-card.json
│   ├── activity-card.wxml
│   └── activity-card.wxss
│
├── activity-list/         # 活动列表组件
│   ├── activity-list.js
│   ├── activity-list.json
│   ├── activity-list.wxml
│   └── activity-list.wxss
│
└── empty-state/         # 空状态组件
    ├── empty-state.js
    ├── empty-state.json
    ├── empty-state.wxml
    └── empty-state.wxss
```

### 4.2 组件使用
在页面中注册使用：
```json
{
  "usingComponents": {
    "activity-card": "/components/activity-card/activity-card",
    "empty-state": "/components/empty-state/empty-state"
  }
}
```

## 五、组件库规范

### 5.1 优先使用 tdesign-miniprogram
- 文档：https://tdesign.tencent.com/miniprogram/getting-started

### 5.2 使用示例
```json
{
  "usingComponents": {
    "t-button": "tdesign-miniprogram/button/button",
    "t-cell": "tdesign-miniprogram/cell/cell",
    "t-toast": "tdesign-miniprogram/toast/toast"
  }
}
```

### 5.3 无对应组件时
- 先考虑用基础组件组合
- 确实需要再自定义开发
