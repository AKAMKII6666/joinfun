## 3. 技术方案

### 3.1 登录流程

```
wx.login() 获取 code
    ↓
wx.request() 发送 code 到后端
    ↓
后端返回 token + 用户信息
    ↓
存入 wx.setStorage()
    ↓
跳转首页
```

### 3.2 接口规范

**请求**
```
POST http://8.152.100.50:10001/account/login
Content-Type: application/json

{
  "code": "xxx"
}
```

**响应（预期）**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "xxx",
    "userInfo": {
      "id": 1,
      "nickname": "xxx",
      "avatar": "xxx"
    }
  }
}
```

### 3.3 路由守护规则

| 页面 | 是否需要登录 | 说明 |
|------|-------------|------|
| 首页 `index` | ❌ 免登录 | 展示活动列表 |
| 活动详情 `activity-detail` | ❌ 免登录 | 查看活动信息 |
| 活动详情-报名按钮 | ✅ 需要登录 | 点击报名需登录 |
| 发布活动 `publish` | ✅ 需要登录 | 创建/编辑活动 |
| 我发起的 `my-activities` | ✅ 需要登录 | 管理我的活动 |
| 我的 `profile` | ✅ 需要登录 | 用户中心 |
| 登录页 `login` | ❌ 免登录 | 登录页面 |

### 3.4 路由守护实现方式

**方式**：在页面 onLoad 中检查登录状态，未登录则跳转登录页

```javascript
// 路由守护工具
const checkLogin = () => {
  const token = wx.getStorageSync('token');
  if (!token) {
    wx.redirectTo({ url: '/pages/login/login' });
    return false;
  }
  return true;
};
```
