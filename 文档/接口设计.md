# 组局小程序 - API 接口设计文档

## 一、接口规范

### 1.1 基础信息

- **协议**：HTTPS
- **数据格式**：JSON
- **字符编码**：UTF-8
- **时间格式**：ISO 8601（`YYYY-MM-DD HH:mm:ss`）

### 1.2 响应格式

**成功响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**失败响应**：
```json
{
  "code": 1001,
  "message": "错误描述",
  "data": null
}
```

### 1.3 通用错误码

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 未授权/登录失效 |
| 1003 | 权限不足 |
| 1004 | 资源不存在 |
| 1005 | 操作失败 |
| 1006 | 频率限制 |

### 1.4 认证方式

- **header**: `Authorization: Bearer <token>`
- 未登录接口无需 token
- 需登录接口返回 1002 错误时，前端调用微信登录刷新 token

## 二、认证模块

### 2.1 微信登录

**接口**：`POST /api/auth/wechat/login`

**描述**：微信小程序登录，首次登录自动注册。

**请求参数**：
```json
{
  "code": "string",  // 微信登录 code（必填）
  "encryptedData": "string",  // 用户数据（可选）
  "iv": "string"  // 加密向量（可选）
}
```

**响应数据**：
```json
{
  "token": "string",  // JWT token
  "expiresIn": 7200,  // 过期时间（秒）
  "userInfo": {
    "id": 10001,
    "openid": "xxx",
    "nickname": "张三",
    "avatarUrl": "https://...",
    "isNewUser": true
  }
}
```

**说明**：
- 后端使用 code 调用微信接口获取 openid/session_key
- 根据 openid 判断是否为新用户
- token 有效期 2 小时，前端需处理刷新逻辑

---

### 2.2 刷新 Token

**接口**：`POST /api/auth/token/refresh`

**描述**：刷新访问 token。

**请求参数**：无（从 header 获取旧 token）

**响应数据**：
```json
{
  "token": "string",
  "expiresIn": 7200
}
```

## 三、用户模块

### 3.1 获取用户信息

**接口**：`GET /api/user/profile`

**描述**：获取当前用户信息。

**请求参数**：无

**响应数据**：
```json
{
  "id": 10001,
  "openid": "xxx",
  "nickname": "张三",
  "avatarUrl": "https://...",
  "modifyCount": 1,
  "modifyLimit": 3,
  "canModify": true
}
```

---

### 3.2 更新用户信息

**接口**：`PUT /api/user/profile`

**描述**：更新用户昵称和头像（年内限 3 次）。

**请求参数**：
```json
{
  "nickname": "string",  // 可选，1-50 字符
  "avatarUrl": "string"  // 可选，图片 URL
}
```

**响应数据**：
```json
{
  "nickname": "张三",
  "avatarUrl": "https://...",
  "modifyCount": 2,
  "remainingCount": 1
}
```

**错误码**：
- 1005: 年内修改次数已达上限

## 四、活动模块

### 4.1 获取活动列表（首页）

**接口**：`GET /api/activities`

**描述**：分页获取活动列表，支持筛选。

**请求参数**（query）：
```
page: number (默认 1)
pageSize: number (默认 20, 最大 50)
status: number (可选：1-进行中，2-已取消，3-已结束)
keyword: string (可选，搜索标题)
sortBy: string (可选：default/latest/hottest)
```

**响应数据**：
```json
{
  "list": [
    {
      "id": 1001,
      "title": "周末爬山活动",
      "coverImage": "https://...",
      "locationName": "香山公园",
      "dates": [
        {
          "date": "2026-03-15",
          "startTime": "09:00",
          "endTime": "17:00"
        }
      ],
      "registrationCount": 25,
      "maxParticipants": 50,
      "organizer": {
        "nickname": "李四",
        "avatarUrl": "https://..."
      },
      "status": 1
    }
  ],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 156,
    "hasMore": true
  }
}
```

---

### 4.2 获取活动详情

**接口**：`GET /api/activities/:id`

**描述**：获取单个活动详细信息。

**请求参数**：
- path: `id` (活动 ID)

**响应数据**：
```json
{
  "id": 1001,
  "title": "周末爬山活动",
  "coverImage": "https://...",
  "description": "活动说明内容...",
  "detailImages": ["https://...", "https://..."],
  "guideImages": ["https://...", "https://..."],
  "locationName": "香山公园",
  "locationAddress": "北京市海淀区...",
  "latitude": 39.986942,
  "longitude": 116.187654,
  "contactPhone": "13800138000",
  "maxParticipants": 50,
  "registrationCount": 25,
  "viewCount": 320,
  "status": 1,
  "dates": [
    {
      "date": "2026-03-15",
      "startTime": "09:00",
      "endTime": "17:00"
    }
  ],
  "organizer": {
    "id": 10002,
    "nickname": "李四",
    "avatarUrl": "https://..."
  },
  "myRegistrationStatus": 0,  // 0-未报名，1-已报名，2-已请假
  "recentRegistrations": [  // 最新报名用户（前 10 个）
    {
      "userId": 10003,
      "nickname": "王五",
      "avatarUrl": "https://...",
      "status": 1
    }
  ]
}
```

**说明**：
- 访问接口自动增加 `viewCount`
- `myRegistrationStatus` 用于判断当前用户报名状态

---

### 4.3 创建活动

**接口**：`POST /api/activities`

**描述**：发布新活动。

**请求参数**：
```json
{
  "title": "string",  // 必填，1-40 字符
  "coverImage": "string",  // 可选
  "maxParticipants": 999,  // 可选，默认 999
  "locationName": "string",  // 可选
  "locationAddress": "string",  // 可选
  "latitude": 39.986942,  // 可选
  "longitude": 116.187654,  // 可选
  "contactPhone": "13800138000",  // 可选，11 位
  "description": "string",  // 可选，1000 字内
  "detailImages": ["https://..."],  // 可选，最多 12 张
  "guideImages": ["https://..."],  // 可选，最多 12 张
  "dates": [  // 必填，至少一个日期
    {
      "date": "2026-03-15",
      "startTime": "09:00",
      "endTime": "17:00"
    }
  ]
}
```

**响应数据**：
```json
{
  "id": 1001
}
```

**错误码**：
- 1001: 参数错误（如日期格式不对、图片超数量）
- 1002: 未登录

---

### 4.4 更新活动

**接口**：`PUT /api/activities/:id`

**描述**：编辑已发布的活动（仅发起人可操作）。

**请求参数**：同 4.3

**响应数据**：
```json
{
  "id": 1001,
  "updatedAt": "2026-03-11 13:00:00"
}
```

**错误码**：
- 1003: 权限不足（非发起人）
- 1004: 活动不存在

---

### 4.5 删除活动

**接口**：`DELETE /api/activities/:id`

**描述**：删除活动（仅发起人可操作）。

**请求参数**：
- path: `id`

**响应数据**：
```json
{
  "success": true
}
```

**说明**：
- 软删除：将活动状态改为"已取消"，封面图加水印
- 已报名用户不受影响，但无法继续报名

---

### 4.6 复用活动（前端行为）

**说明**：复用活动无需单独的后端接口，前端实现即可。

**前端流程**：
1. 用户在"我发起的"列表点击某活动的"复用"按钮
2. 前端调用 `GET /api/activities/:id` 获取原活动详情
3. 跳转到"发布新活动"表单页，自动填充原活动数据
4. **清空日期时间字段**，默认填写当天日期
5. 用户可修改任意字段后点击提交
6. 前端调用 `POST /api/activities` 创建新活动

**响应数据**：同 4.3 创建活动

## 五、报名模块

### 5.1 报名活动

**接口**：`POST /api/activities/:id/register`

**描述**：报名参加活动。

**请求参数**：
- path: `id` (活动 ID)

**响应数据**：
```json
{
  "success": true,
  "registrationId": 5001
}
```

**错误码**：
- 1002: 未登录
- 1004: 活动不存在
- 1005: 报名失败（已满员/活动已结束/重复报名）

---

### 5.2 取消报名（请假）

**接口**：`POST /api/activities/:id/leave`

**描述**：已报名用户请假。

**请求参数**：
- path: `id` (活动 ID)

**响应数据**：
```json
{
  "success": true
}
```

**错误码**：
- 1002: 未登录
- 1004: 活动不存在
- 1005: 操作失败（未报名/活动已结束）

---

### 5.3 获取报名列表

**接口**：`GET /api/activities/:id/registrations`

**描述**：获取活动的报名用户列表。

**请求参数**：
- path: `id` (活动 ID)
- query:
```
page: number (默认 1)
pageSize: number (默认 20)
status: number (可选：1-已报名，2-已请假)
```

**响应数据**：
```json
{
  "list": [
    {
      "userId": 10003,
      "nickname": "王五",
      "avatarUrl": "https://...",
      "status": 1,
      "registeredAt": "2026-03-10 15:30:00"
    }
  ],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 25,
    "hasMore": true
  }
}
```

## 七、浏览记录模块

### 7.1 记录浏览

**接口**：`POST /api/activities/:id/view`

**描述**：记录用户浏览活动（用于"最近浏览"功能）。

**请求参数**：
- path: `id` (活动 ID)

**响应数据**：
```json
{
  "success": true
}
```

**说明**：
- 前端在用户打开活动详情时调用
- 同一用户同一活动只保留最新浏览记录
- 未登录用户不记录

## 八、文件上传模块

### 8.1 上传图片

**接口**：`POST /api/upload/image`

**描述**：上传图片到对象存储。

**请求参数**（multipart/form-data）：
```
file: File (必填，图片文件)
type: string (可选：cover/detail/guide/avatar，用于区分目录)
```

**响应数据**：
```json
{
  "url": "https://oss.example.com/joinfun/cover/xxx.jpg",
  "filename": "xxx.jpg",
  "size": 102400
}
```

**说明**：
- 建议限制单张图片大小（如 5MB）
- 自动压缩/裁剪（可选）
- 返回公网可访问的 URL

## 九、WebSocket 实时通知（可选）

### 9.1 连接 WebSocket

**端点**：`wss://api.example.com/ws`

**描述**：建立 WebSocket 连接接收实时通知。

**连接参数**：
```
token: <JWT token>  // 通过 query 参数传递
```

**推送消息类型**：

**有人报名活动**：
```json
{
  "type": "NEW_REGISTRATION",
  "data": {
    "activityId": 1001,
    "activityTitle": "周末爬山活动",
    "userId": 10003,
    "nickname": "王五",
    "time": "2026-03-11 13:00:00"
  }
}
```

**活动信息变更**：
```json
{
  "type": "ACTIVITY_UPDATED",
  "data": {
    "activityId": 1001,
    "activityTitle": "周末爬山活动",
    "changeType": "TIME_CHANGE",  // TIME_CHANGE/INFO_CHANGE/CANCELLED
    "time": "2026-03-11 13:00:00"
  }
}
```

## 十、接口调用流程示例

### 10.1 用户完整流程

```
1. 小程序启动
   → 检查本地 token
   → 无效则调用微信登录 (2.1)
   → 存储 token

2. 进入首页
   → 获取活动列表 (4.1)
   → 获取我参与的活动 (6.1)
   → 获取最近浏览 (7.2)

3. 浏览活动详情
   → 获取活动详情 (4.2)
   → 记录浏览 (7.1)

4. 报名活动
   → 报名 (5.1)
   → 刷新活动详情 (4.2) 查看状态

5. 发布活动
   → 上传图片 (8.1) × N
   → 创建活动 (4.3)
```

### 10.2 发起人管理流程

```
1. 进入"我发起的"
   → 获取我发起的活动 (6.2)

2. 编辑活动
   → 更新活动 (4.4)
   → 通知报名用户 (WebSocket 9.1 或站内信)

3. 查看报名情况
   → 获取报名列表 (5.3)

4. 删除活动
   → 删除活动 (4.5)
```

## 十一、安全与性能

### 11.1 安全措施

- **HTTPS**：所有接口强制 HTTPS
- **Token 验证**：需登录接口校验 token
- **参数校验**：严格校验输入参数（长度、格式、类型）
- **频率限制**：单 IP/单用户请求频率限制
- **SQL 注入防护**：使用参数化查询
- **XSS 防护**：输出内容转义

### 11.2 性能优化

- **分页**：列表接口必须分页
- **缓存**：活动详情、用户信息可缓存（Redis）
- **CDN**：图片/静态资源走 CDN
- **数据库索引**：按查询场景建立索引
- **异步处理**：浏览计数、通知等非关键操作异步
